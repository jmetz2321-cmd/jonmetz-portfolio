Build a Personal Website with Cursor + GitHub Pages

No prior web development experience required — Cursor's AI handles the code while you focus on content and design decisions.

The Cursor-First Workflow

A static site can be built entirely through conversation with Cursor:

  1. Start from a prompt and an empty folder — open a new workspace in Cursor and prompt it to create index.html, styles.css, and script.js. Use follow-up prompts to shape the design.
  2. Iteratively refine content and layout — edit HTML via natural-language instructions. Adjust CSS for your visual theme. Update JS for interactive behavior.
  3. Preview locally — run python3 -m http.server 8765 and check layout, spacing, and tap targets.
  4. Create a Git repo and push to GitHub — initialize git, commit, create a GitHub repo, and push main.
  5. Enable GitHub Pages — turn on Pages in repo settings. From here on: edit → preview → commit → push → live in ~2 minutes.

1. Prerequisites

  • A GitHub account with SSH or HTTPS access
  • git — verify with git --version
  • Python 3 — verify with python3 --version
  • Cursor IDE installed

2. Create (or clone) the repository

Option A: Start from scratch

mkdir my-site && cd my-site
git init

Option B: Clone an existing repo

git clone git@github.com:<your-username>/<your-repo>.git
cd <your-repo>

A typical project structure:

<your-repo>/ ├── index.html ├── styles.css ├── script.js ├── favicon.svg ├── images/ │ └── (your images here) └── README.md

3. Open the project in Cursor

  1. Open Cursor.
  2. File → Open Folder… and select your project folder.
  3. Cursor will index the files. Use Agent mode or inline edits to modify any file with natural language.

4. Run a local preview server

python3 -m http.server 8765
  • Desktop: open http://localhost:8765/
  • Mobile (same Wi-Fi): use your computer's local IP, e.g. http://192.168.1.42:8765/

5. Editing the site in Cursor

5.1 HTML (index.html)

  • Hero & headline: Your name/title, a one-liner, and a primary CTA.
  • Core sections: About, Projects/Work, Skills, Experience, Contact.
  • Social proof: Testimonials, client logos, ratings, or achievements.

5.2 CSS (styles.css)

  • Theme variables (colors, fonts) defined in :root.
  • A wrapper class controls max-width and horizontal padding.
  • Media queries handle mobile layout and safe-area padding.

5.3 JavaScript (script.js)

  • Scroll-reveal animations via IntersectionObserver
  • Stat/number counter animations (gated behind prefers-reduced-motion)
  • Mobile hamburger menu toggle
  • Active nav-link highlighting on scroll

6. Commit your changes

git status
git add index.html styles.css script.js
git commit -m "Update hero section and add projects"

To unstage a file:

git restore --staged <file>

7. Push to GitHub

git push

8. Configure GitHub Pages (one-time)

  1. Go to your repo on GitHub.
  2. Click Settings → Pages.
  3. Under Source, select Deploy from a branch, branch: main, folder: / (root).
  4. Click Save.

After 1–3 minutes your site is live at:

https://<your-username>.github.io/<your-repo>/

9. Updating the live site (quick path)

  1. Edit in Cursor.
  2. Preview at http://localhost:8765/.
  3. Commit and push:
git add .
git commit -m "Short description of change"
git push

Wait ~1–3 minutes for GitHub Pages to rebuild.

10. Recommended: Feature Branch + PR Workflow

For any meaningful update, use a feature branch and pull request instead of pushing directly to main.

Why this matters

  • Protects your live site. Changes on a feature branch don't affect main.
  • Enables code review. The PR diff shows exactly what changed.
  • Creates a history. Each PR is a self-contained record of what changed and why.

The steps

  1. Create a feature branch: 
    git checkout -b feature/my-change
  2. Tell Cursor what to change in plain English.
  3. Preview locally at http://localhost:8765/.
  4. Commit and push to the feature branch: 
    git add .
git commit -m "feat: describe the change"
git push -u origin feature/my-change
  5. Open a pull request: 
    gh pr create --title "feat: describe the change" \
  --body "Summary of what changed." --base main
  6. Review the PR on GitHub.
  7. Merge — GitHub Pages rebuilds automatically.

Using Cursor shortcut commands

With Cursor skills configured, the workflow simplifies to:

Edit in Cursor → Preview → /commit → /pr → Review → /deploy

11. Setting Up /commit, /pr, and /deploy as Global Skills

The shortcut commands are powered by Cursor Skills — markdown instruction files that teach Cursor reusable workflows. Setting them up at the user level means they work in every project you open.

Install the gh CLI

# Download and install (no Homebrew/sudo required)
curl -sL https://github.com/cli/cli/releases/latest/download/gh_*_macOS_arm64.zip -o /tmp/gh.zip
unzip -o /tmp/gh.zip -d /tmp/gh
mkdir -p ~/bin
cp /tmp/gh/*/bin/gh ~/bin/gh
chmod +x ~/bin/gh
echo 'export PATH=$HOME/bin:$PATH' >> ~/.zshrc
source ~/.zshrc

# Authenticate with GitHub
gh auth login

Verify with gh auth status.

Where to put the skill files

LocationScope
<project>/.cursor/skills/Only that one project
~/.cursor/skills-cursor/Every project you open

Install them at the user level to avoid recreating per repo:

~/.cursor/skills-cursor/ ├── commit/ │ └── SKILL.md ├── pr/ │ └── SKILL.md └── deploy/ └── SKILL.md

Skill 1: /commit

Create ~/.cursor/skills-cursor/commit/SKILL.md:

What it does: Stages changes, creates a conventional commit, and pushes to a feature branch. Never pushes to main directly.

Workflow: Check status → ensure feature branch → git add -A → generate commit message → git commit → git push -u origin

---
name: commit
description: Stages changes, creates a conventional commit,
  and pushes to a feature branch.
---

# Commit and Push to Feature Branch

## Workflow
1. Run `git status` to see what's changed.
2. If on `main`, create a feature branch:
   `git checkout -b feature/<short-description>`
3. Stage: `git add -A`
4. Generate commit message (feat:, fix:, chore:, docs:)
   from `git diff --cached`.
5. Commit: `git commit -m "[message]"`
6. Push: `git push -u origin $(git branch --show-current)`
7. Report hash, message, branch. Remind to run `/pr`.

Skill 2: /pr

Create ~/.cursor/skills-cursor/pr/SKILL.md:

What it does: Creates a pull request from the current feature branch against main.

Workflow: Verify committed + pushed → check not on main → gh pr create → share URL

---
name: pr
description: Creates a pull request for the current branch.
---

# Create Pull Request

## Workflow
1. Ensure changes are committed and pushed.
2. Verify not on main.
3. Check for existing PR: `gh pr view --json url`
4. Create: `gh pr create --title "<title>"
   --body "<body>" --base main`
5. Report the PR URL. User can review with
   `gh pr diff` or run `/deploy` when ready.

Skill 3: /deploy

Create ~/.cursor/skills-cursor/deploy/SKILL.md:

What it does: Merges the current PR into main and triggers a GitHub Pages deploy.

Workflow: Find PR → verify checks → gh pr merge --merge --delete-branch → switch to main → report live URL

---
name: deploy
description: Merges the current PR to main and deploys.
---

# Deploy to Production

## Workflow
1. Find PR: `gh pr view --json number,title,state,url`
2. Verify checks: `gh pr checks`
3. Merge: `gh pr merge --merge --delete-branch`
4. Switch: `git checkout main && git pull origin main`
5. Report merge, share live URL, note Pages takes
   ~1-3 minutes to rebuild.

The workflow in practice

You: "Add a new checkout prototype" → Cursor builds it

You: /commit → feature branch created, committed, pushed

You: /pr → pull request opened on GitHub

You: (review the PR diff)

You: /deploy → merged to main, site goes live

No manual git commands, no switching to the browser, no recreating workflows per repo.

12. Optional: Use as your root GitHub Pages site

By default, project sites live at https://<your-username>.github.io/<your-repo>/. For the shorter root URL, create a repo named exactly <your-username>.github.io:

  1. Create the repo.
  2. Copy your site files into the repo root.
  3. Enable Pages (branch: main, folder: /).

13. Workflow summary

  1. Create or clone a repo and open it in Cursor.
  2. Run python3 -m http.server 8765 to preview locally.
  3. Use Cursor to edit files with natural language.
  4. Verify changes in the browser (desktop + mobile).
  5. git add . && git commit -m "message" && git push
  6. Wait ~2 minutes for GitHub Pages to redeploy.
  7. Visit your live URL and confirm.
